# AI 교육격차 지도 MVP

교육 공공데이터 AI 활용대회용 MVP입니다.

- GitHub: <https://github.com/SeungMin-Park-psm1757/ai_education_gapmap>
- 시연 시작 화면: `/map`
- 현재 버전: 공공데이터 + 임의 학교데이터 결합 모드

## 한 줄 설명

AI 교육격차 지도는 공공데이터와 임의 학교데이터를 결합해 학교별 AI 교육 지원 소요를 확인하고, 교육청이 예산·연수·프로그램 지원을 어디에 먼저 투입할지 돕는 의사결정 MVP입니다.

## 왜 필요한가

교육청은 한정된 예산과 인력으로 여러 학교를 지원해야 합니다. 기존 학교정보 조회만으로는 어느 학교에 디지털 기반, 교원 연수, AI·SW 프로그램, 지역 연계를 먼저 검토해야 하는지 한눈에 보기 어렵습니다.

이 MVP는 흩어진 공공데이터와 공모전 목적의 임의 학교데이터를 결합해 학교별 `지원 소요 지수`를 만들고, 지도와 상세 보고서에서 행정 조치 방안까지 연결합니다.

## 현재 분석 범위

- 지역: 서울특별시 노원구
- 대상: 초·중·고 및 공시 기반 비교가 가능한 학교 98개
- 익명화: 학교 공공데이터는 활용했으나 명칭과 위치는 익명화
- 제외: 학력인정 평생교육시설 2개교
- 제외 이유: 특수목적 성격의 학교로 일반 초·중·고 지원 소요 비교에 함께 넣으면 해석이 왜곡될 수 있음

## 제출용 익명화

UI는 실제 학교명, 주소, 전화번호, 홈페이지, 구체 지역명을 표시하지 않습니다. 학교명은 `A초`, `B중`, `C고` 형식으로 바뀌고, 지역은 `서울 A권역`으로 표시됩니다. 데이터 출처 문서와 산식 설명은 공공데이터 활용 사실을 유지합니다.

익명화 모드의 지도는 실제 노원구 지도를 배경으로 사용하되, 학교 마커는 실제 좌표가 아닌 임의 분포로 표시합니다.

`.env.example` 기본값은 제출용 설정입니다.

```env
NEXT_PUBLIC_DEMO_ANONYMIZE=true
NEXT_PUBLIC_DATA_VERSION=v3
```

## 핵심 지표

`지원 소요 지수`는 점수가 높을수록 우수하다는 뜻이 아닙니다. AI교육을 위한 우선 지원이 필요하다는 뜻입니다.

총점은 100점 만점이며 다음 5개 영역을 반영합니다.

| 영역 | 가중치 | 의미 |
| --- | ---: | --- |
| 교육 수요 | 25 | 학생 규모, 학급 규모 등 지원 수요 |
| 교원·운영 여건 | 20 | 교원 배치와 운영 여건 보강 필요 |
| 디지털·학습공간 기반 | 20 | 기기, 무선망, 학습공간 관련 보강 필요 |
| AI·SW 학습기회 | 20 | 방과후, 동아리, AI·SW 프로그램 접근성 |
| 지역 지원 접근성 | 15 | 학교 밖 프로그램과 지역 자원 연계 필요 |

데이터 신뢰도는 총점에 넣지 않고 A/B/C 배지로 분리합니다. 신뢰도 C가 있을 경우 점수 순위에 섞지 않고 `현장 확인 우선`으로 별도 표시합니다.

## 주요 화면

| 경로 | 역할 |
| --- | --- |
| `/` | 서비스 목적, 분석 학교 수, 지원 구간 요약 |
| `/map` | 우선지원 필요 TOP 3, 지도 기반 지원 소요 분포, 데이터 근거 |
| `/priorities` | 학교별 지원 소요 지수, 대표 보강 영역, 조치 방안 |
| `/schools/[id]` | 학교 상세 보고서, 점수산정 기준 및 결과, 행정 조치 방안 |
| `/data` | 데이터 출처, 산식, 직접지표와 대체지표, 임의 학교데이터 설명 |

별도 3분 시연 화면에 의존하지 않고, 격차지도(`/map`)만 봐도 점수와 지원 우선 대상을 바로 이해하도록 동선을 단순화했습니다.

## 데이터 구성

### 실제 공공데이터

- NEIS 학교 기본정보
- 학교알리미 공시자료
- 교육통계 및 교육청 공개자료
- 학교 표준자료, 시설·프로그램 공개자료

지도 마커는 실제 좌표가 아니라 익명화를 위한 가상 배치입니다.

### 임의 학교데이터

v3는 공모전 목적의 임의 학교데이터를 공공데이터와 결합해 산출합니다. 실제 운영 단계에서는 학교가 제출한 AI 디지털교과서(AIDT), 학습관리시스템(LMS), 교원 연수, 기기 접근성, AI·SW 프로그램 운영 자료로 임의 데이터를 대체할 수 있습니다.

임의 학교데이터 항목:

- AI 디지털교과서(AIDT) 접속 안정성
- 학습관리시스템(LMS) 사용 지속성
- 교원 연수 이수
- 기기 접근성
- AI·SW 프로그램 운영
- 외부 AI프로그램 접근성

## 실행 방법

```bash
npm install
cp .env.example .env.local
npm run prepare:data
npm run dev
```

로컬 실행 후 다음 주소에서 확인합니다.

```text
http://localhost:3000/map
```

```bash
npm run check:copy
npm run typecheck
npm run build
```

## 대회 발표 포인트

- 개별 학교의 우열을 가르는 서비스가 아니라 지원 소요를 찾는 도구입니다.
- 공공데이터와 임의 학교데이터를 결합해 지원 소요 지수를 산출합니다.
- 지도에서 우선지원 필요 TOP 3를 바로 확인하고, 상세 보고서에서 예산·연수·프로그램·자료 보완 조치로 연결합니다.
- 실제 운영 단계에서는 임의 학교데이터를 학교 제출자료나 교육청 시스템 연계자료로 대체할 수 있습니다.

# 최종 제출 데이터 정책

## 1. 원칙

AI 교육격차 지도 MVP의 제출용 본체는 공개 교육 공공데이터 기반 1단계 지원 소요 지수이다. 확장진단 시나리오 데이터는 추가자료가 제공될 경우의 분석 구조를 보여주는 예시이며, 메인 지원 소요 지수에는 반영하지 않는다.

핵심 문장:

> 임의 학교데이터는 실제 학교 제출자료가 아니라, 학교·교육청 추가자료가 제공될 경우의 확장진단 구조를 보여주는 시나리오 데이터입니다. 제출용 메인 지수는 공개 공공데이터 기반으로 제시합니다.

## 2. 메인 지수에 반영한 데이터

| 데이터 | 출처 | 반영 여부 | 사용 목적 |
| --- | --- | --- | --- |
| NEIS 학교 기본정보 | NEIS 교육정보 개방 포털 | 예 | 학교 식별, 학교급, 교육지원청, 주소 대조 |
| 학교알리미 공시자료 | 학교알리미 | 예 | 학생 수, 교원 수, 학급 수, 시설·프로그램 지표 |
| 공공데이터포털 학교 표준자료 | 공공데이터포털 | 예 | 기관 구분과 주소 대조 |
| 교육통계·교육청 공개자료 | 교육통계·교육청 공개자료 | 예 또는 보조 | 시설, 정보화, 프로그램 관련 보강 신호 |

## 3. 메인 지수에 반영하지 않은 데이터

| 데이터 | 반영 여부 | 용도 |
| --- | --- | --- |
| 확장진단 시나리오 데이터 | 아니오 | 학교·교육청 추가자료 제공 시 가능한 분석 구조 설명 |

시나리오 항목:

- AI 디지털교과서(AIDT) 접속 안정성
- 학습관리시스템(LMS) 사용 지속성
- 교원 연수 이수
- 기기 접근성
- AI·SW 프로그램 운영
- 외부 AI프로그램 접근성

## 4. 익명화 정책

| 항목 | 처리 방식 |
| --- | --- |
| 학교명 | `A초`, `B중`, `C고` 형식 |
| 주소 | `서울 A권역` |
| 교육지원청 | `서울 A권역 교육지원청` |
| 전화번호 | 제출용 UI에서 표시하지 않음 |
| 홈페이지 | 제출용 UI에서 표시하지 않음 |
| 지도 마커 | 실제 좌표가 아닌 가상 배치 |

## 5. 개인정보·식별정보 처리

제출용 화면은 개별 학교 식별정보를 노출하지 않는다. 데이터 출처와 산식 설명에서는 공공데이터 활용 사실을 유지하되, UI에서는 학교명과 위치를 익명화한다.

## 6. 부정행위 방지 원칙

- 시나리오 데이터는 메인 지수 산출값처럼 표현하지 않는다.
- 시나리오 데이터는 확장진단 구조 설명에만 사용한다.
- 발표와 README에서 공개 공공데이터 기반 지수와 시나리오 데이터를 분리해 설명한다.
- AI가 최종 결론을 내린 것처럼 표현하지 않는다.

# AI 교육격차 지도 v3 프로그램 구성

작성일: 2026-05-28  
버전: v3-public 제출용 대시보드  
기본 시작 화면: `/`

## 1. 서비스 개요

`AI 교육격차 지도`는 공개 교육 공공데이터를 결합해 학교별 AI 교육 지원 소요를 확인하는 의사결정 대시보드이다.

핵심 목적은 학교를 평가하거나 서열화하는 것이 아니라, 교육청이 예산, 연수, 프로그램, 자료 보완을 어디에 먼저 검토할지 돕는 것이다.

핵심 문장:

> 점수가 높을수록 AI 교육 지원소요가 큽니다.

## 2. 제출용 기본 정책

| 항목 | 현재 설정 |
| --- | --- |
| 기본 데이터 모드 | `v3-public` |
| 메인 지수 | 공개 공공데이터 기반 1단계 지원 소요 지수 |
| 익명화 | 기본 적용 |
| 학교명 표시 | `A초`, `B중`, `C고` 형식 |
| 지역 표시 | `서울 A권역` 형식 |
| 지도 위치 | 실제 지도 배경 + 학교 마커 가상 배치 |
| 확장진단 시나리오 | 메인 지수 미반영 |

`.env.example` 제출용 기본값:

```env
NEXT_PUBLIC_DEMO_ANONYMIZE=true
NEXT_PUBLIC_DATA_VERSION=v3-public
NEXT_PUBLIC_INCLUDE_SCENARIO_DATA=false
```

## 3. 분석 범위

| 구분 | 내용 |
| --- | --- |
| 대상 지역 | 서울 A권역 모델 |
| 분석 학교 | 98개 |
| 제외 학교 | 학력인정 평생교육시설 2개교 |
| 제외 이유 | 일반 초·중·고 지원 소요 비교에 포함하면 해석 왜곡 가능 |
| 제출 화면 | 학교명, 주소, 전화번호, 홈페이지, 구체 지역명 미노출 |

## 4. 현재 산출 결과

공개 공공데이터 기반 1단계 지수 기준이다.

| 구분 | 학교 수 | 의미 |
| --- | ---: | --- |
| 우선지원 필요 | 7 | 공개자료상 지원 소요가 큰 학교군 |
| 지원여부 검토 | 71 | 보강 필요 항목을 검토할 학교군 |
| 일반 모니터링 | 20 | 현재 자료상 긴급 지원 신호가 크지 않은 학교군 |
| 현장 우선확인 필요 | 0 | 데이터 신뢰도 C로 점수보다 자료 확인이 우선인 학교군 |
| 합계 | 98 | 제출용 분석 대상 |

## 5. 핵심 지표 구조

`지원 소요 지수`는 100점 만점이다. 점수가 높을수록 우수한 학교가 아니라, AI 교육 지원이 더 필요할 가능성이 큰 학교를 뜻한다.

| 영역 | 가중치 | 해석 |
| --- | ---: | --- |
| 교육 수요·취약 여건 | 25 | 학생 규모, 학급 규모 등 지원 수요 |
| 교원·운영 여건 | 20 | 교원 배치와 운영 여건 보강 필요 |
| 디지털·학습공간 기반 | 20 | 기기, 무선망, 학습공간 관련 보강 필요 |
| AI·SW 학습기회 | 20 | 방과후, 동아리, AI·SW 프로그램 접근성 |
| 지역 지원 접근성 | 15 | 학교 밖 프로그램과 지역 자원 연계 필요 |

데이터 신뢰도는 총점에 섞지 않고 A/B/C 배지로 분리한다. 신뢰도 C는 점수가 높더라도 `우선지원 필요`에 섞지 않고 `현장 우선확인 필요`로 분리한다.

## 6. 데이터 구성

### 6-1. 메인 지수 반영 데이터

| 데이터 | 활용 목적 | 메인 지수 반영 |
| --- | --- | --- |
| NEIS 학교 기본정보 | 학교 식별 기준, 학교급, 기본 현황 정규화 | 예 |
| 학교알리미 공시자료 | 학생·교원·운영 여건 관련 공개 지표 | 예 |
| 공공데이터포털 학교 표준자료 | 학교 기본 속성 및 공개 기준 보완 | 예 |
| 교육통계·교육청 공개자료 | 학교급·지역 단위 비교 기준 보완 | 예 |
| 시설·프로그램 공개자료 | 학습공간, 프로그램 접근성 보완 | 예 |

### 6-2. 확장진단 시나리오 데이터

아래 항목은 메인 지원 소요 지수에 반영하지 않는다. 실제 구현 시 학교나 교육청이 추가자료를 제공하면 어떤 구조로 정밀 진단할 수 있는지 보여주는 가정 데이터명이다.

| 항목 | 용도 |
| --- | --- |
| AI 디지털교과서(AIDT) 접속 안정성 | 접속 장애, 서비스 안정성 확인 |
| 학습관리시스템(LMS) 사용 지속성 | 학습 플랫폼 활용 지속성 확인 |
| 교원 연수 이수 | AI 활용 수업 준비도 보완 확인 |
| 기기 접근성 | 학생 기기 접근 격차 확인 |
| AI·SW 프로그램 운영 | 학교 내 학습기회 보완 확인 |
| 외부 AI프로그램 접근성 | 지역·외부 자원 연계 가능성 확인 |

## 7. AI 활용 위치

AI는 학교를 최종 판단하지 않는다. 규칙 기반 산식으로 계산한 결과를 해석하고 설명하는 보조 역할로 사용한다.

| AI 활용 위치 | 설명 |
| --- | --- |
| 취약 요인 라벨링 | 점수 구성에서 두드러지는 보강 영역을 자동 라벨링 |
| 유사 학교군 분류 | 유사한 지원 소요 패턴을 가진 학교군 분류 |
| 지원 유형 매칭 | 예산, 연수, 프로그램, 자료 보완 등 조치 방안 연결 |
| 상세 보고서 설명문 생성 보조 | 산출 결과를 행정 문장으로 요약 |
| 제출자료 문구 정리 보조 | 대회 제출용 설명 문구 정돈 |

## 8. 화면 구성

### 8-1. 홈 `/`

역할: 서비스 목적과 제출용 전제를 가장 먼저 보여주는 기본 화면.

주요 구성:

- 서비스명과 한 줄 설명
- `공개자료 기반`, `학교명 익명화`, `지원 소요 진단`, `확장진단 시나리오 분리` 배지
- 분석 학교 수, 우선지원 필요, 지원여부 검토, 현장 우선확인 필요 KPI
- 주요 화면 이동 버튼

표현 기준:

- 화면에서는 “AI 교육 지원 소요 진단”을 중심 표현으로 사용한다.
- 확장진단 시나리오가 메인 점수처럼 보이지 않게 한다.

### 8-2. AI 교육 지원 소요 지도 `/map`

역할: 심사위원이 가장 빠르게 지원 소요 분포를 볼 수 있는 화면.

주요 구성:

- 우선지원 필요 TOP 3
- 서울 A권역 지도 배경
- 학교 마커 가상 배치
- 범례: 우선지원 필요, 지원여부 검토, 일반 모니터링, 현장 우선확인 필요
- 데이터 근거 요약
- AI교육여건 지원요소 상세 표

지도 안내:

> 제출용 화면에서는 학교 위치를 가상 배치했습니다.

### 8-3. 지원 우선순위 `/priorities`

역할: 학교별 지원 소요와 조치 방안을 행정 업무 단위로 확인하는 화면.

카드 구성:

- 학교명
- 지원 소요 지수
- 신뢰도
- 대표 보강 영역
- 왜 필요한가
- 조치 방안
- 필요한 추가자료
- 상세 보기

조치 방안 예시:

- 교사의 AI 활용 수업 연수 및 수업설계 컨설팅 우선 배정 검토
- 디지털 학습공간·기기 현황 점검 후 정보화 예산 검토
- 방과후 AI·SW 프로그램 또는 지역 SW교육센터 연계 검토
- 자료 결측 항목 보완을 위한 학교 추가자료 요청

### 8-4. 학교 상세 보고서 `/schools/[id]`

역할: 개별 학교의 점수 근거와 다음 조치 방안을 확인하는 화면.

주요 구성:

- 익명화된 학교명
- 지원 소요 지수
- 신뢰도
- 대표 보강 영역
- 점수산정 기준 및 결과
- 행정 조치 방안
- 확장진단 시나리오 접힘 영역

상세 보고서에서는 점수 자체보다 “왜 이 조치가 필요한가”와 “어떤 추가자료가 필요한가”를 확인하도록 구성한다.

### 8-5. 데이터와 AI 활용 `/data`

역할: 제출 조건인 데이터 출처, 산식, AI 활용 위치, 한계를 설명하는 화면.

주요 섹션:

- 활용 데이터 표
- 전처리·정규화 절차
- 지원 소요 지수 산식
- 직접지표와 대체지표 구분
- 데이터 신뢰도 처리
- AI 활용 위치
- 확장진단 시나리오와 향후 조사 항목
- 한계와 보완 방향

핵심 문장:

> 메인 지원 소요 지수는 공개 교육 공공데이터 기반 1단계 진단입니다. 확장진단 시나리오 데이터는 메인 지수에 반영하지 않습니다.

## 9. 용어 기준

| 사용 표현 | 의미 |
| --- | --- |
| 지원 소요 지수 | 공개자료상 AI 교육 지원이 필요한 정도 |
| 지원 필요 신호 | 지원 검토가 필요한 공개자료상 징후 |
| 우선지원 필요 | 지원 소요가 큰 학교군 |
| 지원여부 검토 | 지원 필요사항을 추가로 검토할 학교군 |
| 일반 모니터링 | 공개자료상 긴급 지원 신호가 크지 않은 학교군 |
| 현장 우선확인 필요 | 데이터 부족으로 점수보다 자료 확인이 우선인 학교군 |
| 조치 방안 | 교육청이 검토할 수 있는 행정 조치 |
| 확장진단 시나리오 | 추가자료 제공 시 가능한 정밀 진단 구조 |

화면 문구에서는 우열, 서열, 자동 결정으로 읽힐 수 있는 표현을 사용하지 않는다.

## 12. 실행 및 검증

실행:

```bash
npm install
cp .env.example .env.local
npm run prepare:data
npm run check:data
npm run check:copy
npx tsc --noEmit
NEXT_PUBLIC_DEMO_ANONYMIZE=true NEXT_PUBLIC_DATA_VERSION=v3-public NEXT_PUBLIC_INCLUDE_SCENARIO_DATA=false npm run build
NEXT_PUBLIC_DEMO_ANONYMIZE=true NEXT_PUBLIC_DATA_VERSION=v3-public NEXT_PUBLIC_INCLUDE_SCENARIO_DATA=false npm run start -- -p 3003
```

확인 주소:

```text
http://localhost:3003/
```

검증 기준:

- 실제 학교명, 주소, 전화번호, 홈페이지가 노출되지 않는다.
- 홈이 기본 시작 화면이다.
- `/map`에서 우선지원 필요 TOP 3와 지도 분포가 바로 보인다.
- 점수가 높을수록 AI 교육 지원소요가 크다는 방향이 명확하다.
- 신뢰도 C는 현장 우선확인 필요로 분리된다.
- 확장진단 시나리오 데이터는 메인 지수에 반영되지 않는다.
- `/data`에서 데이터 출처, 산식, AI 활용 위치, 한계가 확인된다.

## 14. 남은 한계

- 공개자료만으로 실제 AI 수업의 질을 확정할 수 없다.
- AI 디지털교과서(AIDT), 학습관리시스템(LMS), 교원 연수 이력 등은 실제 기관 연계 전까지 확장진단 시나리오로만 제시된다.
- 지도 마커는 익명화를 위해 실제 좌표가 아니라 가상 배치이다.

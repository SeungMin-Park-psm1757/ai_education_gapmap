# AI 교육격차 지도 MVP v3 최종 검토 문서

작성 기준: v3 제출용 대시보드  
검토 목적: 영상 없이도 대시보드와 발표자료만으로 문제의식, 데이터 활용, AI 활용, 정책적 효과를 설명할 수 있는지 확인

## 1. 현재 버전 요약

| 항목 | 내용 |
| --- | --- |
| 프로젝트명 | AI 교육격차 지도 MVP |
| 현재 기준 버전 | v3 |
| 기본 데이터 모드 | 공공데이터 + 임의 학교데이터 결합 |

v3는 기존 공개자료 기반 MVP에 학교가 추가자료를 제공했다고 가정한 임의 학교데이터를 결합한 버전이다. 화면 구조는 v2와 거의 동일하게 유지하되, 산출값과 데이터 설명을 v3 기준으로 조정했다.

## 2. 서비스 정의

AI 교육격차 지도는 공공데이터와 학교 데이터를 종합하여 AI 교육여건 활성화를 위한 지원소요를 확인하는 의사결정 MVP이다.

핵심 문구:

> 학교 평가가 아니라 AI 교육 지원 소요 진단

점수가 높을수록 우수한 학교가 아니라, AI교육을 위한 우선 지원이 필요하다는 뜻이다. 서비스의 목적은 학교 서열화가 아니라 교육청이 예산, 연수, 프로그램, 자료 보완을 어디에 먼저 검토할지 돕는 것이다.

## 3. 제출용 익명화 상태

현재 제출용 화면은 익명화가 기본 적용되도록 수정했다.

| 구분 | 처리 방식 |
| --- | --- |
| 실제 학교명 | `A초`, `B중`, `C고` 형식으로 변환 |
| 주소 | `서울 A권역`으로 권역화 |
| 교육지원청 | `서울 A권역 교육지원청`으로 권역화 |
| 전화번호 | 표시하지 않음 |
| 홈페이지 | 표시하지 않음 |
| 지도 마커 | 실제 노원구 지도를 배경으로 쓰되, 학교 위치는 가상 배치 |

안전장치:

- `NEXT_PUBLIC_DEMO_ANONYMIZE=false`라고 명시하지 않는 한 익명화 적용
- `NEXT_PUBLIC_DATA_VERSION`이 비어 있으면 v3 기본 적용
- `.env.example` 기본값은 `NEXT_PUBLIC_DEMO_ANONYMIZE=true`, `NEXT_PUBLIC_DATA_VERSION=v3`

최근 수정 원인:

`/map`은 정적 페이지라서 서버 실행 시점이 아니라 빌드 시점의 환경변수 영향을 받는다. 이전에는 빌드가 비익명 상태로 만들어져 실제 학교명이 노출되었다. 현재는 빌드 기본값과 코드 기본값 모두 익명화 쪽으로 보정했다.

- “학교 평가가 아니라 AI 교육 지원 소요 진단”
- “노원구 학교를 모델로 구축(단, 학교명과 위치 익명화)”
- “임의데이터 구축(공모전 목적)” 표시
- CTA: “AI 교육격차 지도로 확인”

### 격차지도 `/map`

목표:

- 심사위원이 10초 안에 우선지원 필요 학교와 지원소요 분포를 이해
- 실제 노원구 지도 위에 학교를 가상 배치
- 우측 목록과 하단 표로 바로 조치 방안을 확인

현재 반영:

- 우선지원 필요 TOP 목록 표시
- 학교명 익명화
- 지도 우하단에 “학교 위치는 가상의 위치입니다.” 표시
- 학교급은 `초등학교`, `중학교`, `고등학교`, `기타(0학교)` 형식
- 데이터 근거 표에 실제 데이터 활용과 임의데이터 구축을 분리
- `AI교육여건 지원요소 상세` 표의 `제안 조치` 칸 확대

지도 범례:

| 표시 | 의미 |
| --- | --- |
| 우선지원 필요 | 지원 소요 지수가 높아 우선 검토 필요 |
| 지원 필요사항 검토 | 특정 영역 보강 검토 필요 |
| 일반 모니터링 | 현재 자료상 큰 보강 항목은 낮음 |
| 현장 확인 우선 | 신뢰도 C가 있을 때만 조건부 표시 |

### 지원 우선순위 `/priorities`

목표:

- 점수가 높은 학교부터 조치 방안을 확인
- 학교별로 왜 필요한지, 무엇을 할지, 어떤 추가자료가 필요한지 제시

현재 반영:

- 화면 제목: “지원 우선순위”
- 설명: “점수가 높을수록 AI교육을 위한 우선 지원이 필요합니다.”
- `다음 조치` 표현을 `조치 방안`으로 변경
- 카드 구조:
  - 학교명
  - 지원 소요 지수
  - 신뢰도
  - 대표 보강 영역
  - 왜 필요한가
  - 조치 방안
  - 추가자료

### 학교 상세 보고서 `/schools/[id]`

목표:

- 특정 학교가 왜 해당 점수인지 설명
- 점수산정 기준과 행정 조치 방안을 연결
- 개별 학교 식별정보는 제출용 화면에서 제거

현재 반영:

- “학교 상세 보고서”
- “공모전 제출을 위해 익명화되어 개별 학교 식별정보는 표시하지 않습니다.”
- “점수산정 기준 및 결과”
- 표 제목 행: `지수열 / 점수(100점) / 반영 점수 / 판단 이유`
- `총점 산출` 박스 제거
- `2단계 확장진단` 별도 섹션 제거
- 행정 조치 카드 3열 균등 배치

`2단계 확장진단` 제거 이유:

v2에서는 학교·플랫폼 데이터가 나중에 들어올 때의 확장 가능성을 보여주는 용도였다. v3는 이미 임의 학교데이터를 점수에 반영한 버전이므로 별도 확장진단을 남기면 기능 중복처럼 보이고 미완성 인상을 줄 수 있다. 따라서 상세 화면에서는 제거하고, `/data`에서 방법론으로 설명한다.

### 데이터와 AI 활용 `/data`

목표:

- 공공데이터 활용 조건, 출처, 산식, AI 활용 위치를 제출 기준에 맞게 설명
- 임의 학교데이터가 무엇인지, 향후 실제 학교 조사 시 어떻게 넣을지 제시

현재 반영 섹션:

- 활용 데이터 표
- 전처리·정규화 절차
- 지원 소요 지수 산식
- 직접지표와 대체지표
- AI 활용 위치
- 임의 학교데이터와 향후 조사

임의 학교데이터 설명:

- 현 상태: 공모전 목적 임의 학교데이터 98건을 만들어 공공데이터와 결합
- 항목: AI 디지털교과서(AIDT) 접속 안정성, 학습관리시스템(LMS) 사용 지속성, 교원 연수 이수, 기기 접근성, AI·SW 프로그램 운영, 외부 AI프로그램 접근성
- 현재 문구: 해당 항목을 임의 데이터로 조치
- 향후 반영: 학교별 엑셀·CSV를 학교 ID 기준으로 연결하고 항목별 0-100점으로 정규화

## 6. 지수 해석

지원 소요 지수는 역방향 지수이다.

| 점수 방향 | 해석 |
| --- | --- |
| 점수 높음 | AI교육을 위한 우선 지원 필요 |
| 점수 낮음 | 현재 자료상 큰 보강 소요가 낮음 |

현재 화면 용어:

- 우선지원 필요
- 지원 필요사항 검토
- 일반 모니터링
- 현장 확인 우선
- 조치 방안
- 지원 소요 지수
- 지원 필요사항
- 보강 검토 대상

피해야 할 표현:

- 학교 평가
- 부실 학교
- 낮은 학교
- AI가 판단
- 확정 추천
- 우수 학교
- 열악 학교

## 7. AI 활용 위치

AI는 학교를 평가하거나 최종 판단하지 않는다. 명시 산식으로 산출된 결과를 해석하고 설명하는 보조 역할이다.

| 규칙 기반 산식 | 5개 영역 가중치와 신뢰도 분류 계산 |

| 지원 유형 매칭 | 보강 요인에 맞는 조치 방안 연결 |
| 보고서 보조 | 상세 보고서 설명문과 조치 방안 문구 생성 보조 |

## 10. 남은 주의사항

1. 실제 배포 환경에서도 빌드 전에 `NEXT_PUBLIC_DEMO_ANONYMIZE=true`, `NEXT_PUBLIC_DATA_VERSION=v3`가 적용되어야 한다.
2. v3의 학교데이터는 공모전 목적 임의 데이터이므로 발표에서 “실제 학교 제출자료”가 아니라 “운영 구조 검증용 임의 데이터”라고 설명해야 한다.
3. 점수는 학교 평가 점수가 아니라 지원 소요 진단 결과임을 발표 첫 화면과 질의응답에서 반복 확인해야 한다.
4. 실제 운영 단계에서는 학교별 엑셀·CSV 또는 교육청 시스템 연계를 통해 임의 학교데이터를 실제 자료로 교체해야 한다.

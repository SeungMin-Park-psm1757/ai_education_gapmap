# Redteam Response

작성일: 2026-05-24

## 1. 점수 방향 오해 리스크

- 위험: 점수가 높을수록 우수하다고 오해할 수 있다.
- 현재 대응: 화면 표시명을 `지원 소요 지수`로 바꾸고, 지도에 점수 방향 설명을 한 번만 노출했다.
- 남은 한계: 처음 보는 사용자는 숫자만 보고 오해할 수 있다.
- 발표 답변 문장: "이 점수는 성과 점수가 아니라 공개자료상 지원이 먼저 필요한 정도를 보여주는 신호입니다."

## 2. 학교 서열화 리스크

- 위험: 목록과 지도 마커가 학교 줄 세우기로 보일 수 있다.
- 현재 대응: `다음 행정 조치`, `보강 검토`, `현장 확인 우선` 등 실행 중심 표현으로 바꿨다.
- 남은 한계: 숫자형 지수 자체가 비교 인상을 줄 수 있다.
- 발표 답변 문장: "비교의 목적은 학교 간 우열 판단이 아니라 한정된 지원 자원의 배분 순서를 정하는 것입니다."

## 3. 대리지표 취약성

- 위험: 공개자료 대리지표가 실제 수업 변화를 충분히 설명하지 못할 수 있다.
- 현재 대응: `/data`에 직접지표, 대리지표, 추가자료 필요 항목을 분리해 표시했다.
- 남은 한계: AI·SW 학습기회는 동아리와 방과후 자료 등 간접 신호를 포함한다.
- 발표 답변 문장: "1단계는 공개자료로 가능한 1차 신호이고, 실제 운영 데이터가 연결되면 2단계로 정밀화합니다."

## 4. 신뢰도 C 학교 처리

- 위험: 결측이 많은 학교가 높은 점수로 우선 표시될 수 있다.
- 현재 대응: 신뢰도 C는 `현장 확인 우선`으로 분리하고 TOP 3에서 제외한다.
- 남은 한계: 현재 데이터에는 신뢰도 C 사례가 없어 조건부 UI로 준비되어 있다.
- 발표 답변 문장: "자료가 부족한 학교는 점수로 밀어붙이지 않고 현장 확인 대상으로 분리합니다."

## 5. 2단계 확장진단 미연계 리스크

- 위험: AIDT·LMS가 아직 연결되지 않아 미완성처럼 보일 수 있다.
- 현재 대응: `확장진단 대기`와 `확장진단 예시 보기`로 향후 연계 시나리오를 보여준다.
- 남은 한계: 실제 로그 기반 결과는 기관 협력이 필요하다.
- 발표 답변 문장: "현재는 공개자료형 MVP이고, AIDT·LMS 로그가 제공되면 접속 안정성과 사용 지속성을 별도 리포트로 산출합니다."

## 6. 실행 가능성 부족 리스크

- 위험: 점수만 있고 다음 행정 조치가 약하면 현장 활용성이 낮아 보인다.
- 현재 대응: 상세 리포트에 예산 검토, 연수 배정, 프로그램 연계, 현장 확인 카드를 추가했다.
- 남은 한계: 실제 사업명과 예산 항목은 교육청별로 매핑이 필요하다.
- 발표 답변 문장: "지수에서 끝나지 않고 예산, 연수, 프로그램, 현장 확인으로 바로 이어지는 조치 카드를 제공합니다."

## 7. 발표 시연 설득력

- 위험: 별도 시연 화면이 있으면 심사위원이 실제 서비스 화면보다 설명용 화면에 머물 수 있다.
- 현재 대응: v2에서는 `/demo`를 핵심 목록에서 제거하고 `/map`을 첫 시연 화면으로 둔다.
- 남은 한계: 발표자는 지도에서 TOP 3 학교를 고른 뒤 상세 리포트로 들어가는 동선을 짧게 연습해야 한다.
- 발표 답변 문장: "별도 설명 화면보다 격차지도에서 바로 우선 지원 대상과 점수 근거를 보여주고, 상세 리포트에서 다음 행정 조치까지 연결합니다."

# MVP Refinement Report

작성일: 2026-05-24

## 1. 용어 변경 내역

- 화면 표시명을 `지원 소요 지수`로 통일했다.
- 구간명을 `우선 지원`, `보완 검토`, `일반 모니터링`, `현장 확인 우선`으로 정리했다.
- AIDT·LMS 상태 표현은 `확장진단 대기`로 바꿨다.
- 기술 설명에서는 `지원 소요 지수`가 공개자료상 AI 교육 지원이 먼저 필요한 정도라는 점을 명시했다.

## 2. 가치 제안 중심 문구로 전환

- 홈 첫 화면은 한정된 예산과 지원 인력이 먼저 가야 할 학교를 찾는 의사결정 MVP로 설명했다.
- 지도 화면은 10초 안에 우선 지원 학교와 현장 확인 대상을 구분하는 흐름으로 정리했다.
- 방어 문구는 `/data`와 상세 하단의 방법론 영역으로 이동했다.

## 3. 신뢰도 C 처리 방식

- 신뢰도 C는 `현장 확인 우선`으로 분리하는 UI 로직을 추가했다.
- 지도 TOP 3와 우선 확인 목록에서는 신뢰도 C를 제외한다.
- 별도 섹션에서 공개자료가 부족해 점수보다 현장 확인이 먼저 필요하다고 표시한다.
- 현재 노원구 데이터에는 신뢰도 C 학교가 없지만, 데이터가 들어올 경우 즉시 분기된다.

## 4. 실행 가능한 행정 조치 카드

- 학교 상세 리포트에 `다음 행정 조치` 섹션을 추가했다.
- 조치 유형은 `예산 검토`, `연수 배정`, `프로그램 연계`, `현장 확인`이다.
- 학교별 보강 영역에 따라 조치 문구가 달라진다.
- 모든 문장은 확정 지시가 아니라 검토형 문장으로 작성했다.

## 5. 2단계 확장진단 표현 개선

- AIDT·LMS 영역은 `확장진단 대기`로 표시한다.
- 접속·가동, 장애·복구, 사용 지속성, 적용 범위를 별도 진단 항목으로 제시했다.
- `확장진단 예시 보기` 토글을 추가했다.
- 예시 데이터는 1단계 지수에 반영하지 않는다고 명시했다.

## 6. v2 화면 흐름 조정

v1에서는 `/demo`를 별도 설명 화면으로 두었으나, v2에서는 상단 목록과 핵심 화면 목록에서 제거했다.
격차지도 화면만으로도 우선 지원 TOP 3, 지도 분포, 점수 방향 설명이 즉시 보이므로 발표 동선을 `/map` 중심으로 단순화했다.

## 7. 남은 한계

- 현재 지수는 공개자료 기반 1차 신호다.
- 실제 AIDT·LMS 운영 데이터는 아직 연결되지 않았다.
- 교원 연수 이력과 실제 수업 활용률은 추가자료가 있어야 정밀화할 수 있다.
- 현재 데이터에는 신뢰도 C 학교가 없어 현장 확인 분리 UI는 조건부로 검증된다.

## 8. 경연대회 심사기준 대응

- 공공데이터 활용성: NEIS, 학교알리미, 교육통계, 교육청 공개자료를 결합하는 구조를 명확히 했다.
- 문제 해결성: 예산·연수·프로그램 투입 우선순위라는 행정 의사결정 문제에 집중했다.
- AI 확장성: AIDT·LMS 자료 연계 시 운영 안정성 진단으로 확장되는 구조를 제시했다.
- 시연성: `/map`에서 우선 지원 TOP 3와 점수 방향을 바로 보여주고, 상세 리포트로 연결한다.
- 책임성: 데이터 신뢰도를 총점에서 분리하고, 신뢰도 C를 별도 확인 대상으로 처리한다.

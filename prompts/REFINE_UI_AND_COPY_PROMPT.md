너는 교육 공공데이터 AI 활용대회 MVP의 UX 라이터이자 디자이너다.

목표:
기능 추가 없이 화면 문장, 용어, 여백, 카드 구조를 다듬어 심사위원이 10초 안에 이해하게 만든다.

수정 원칙:
- 한 카드에 핵심 문장 1~2개만 둔다.
- 기술 설명은 /data와 /demo에 모은다.
- 홈에서는 서비스 가치와 CTA만 보여준다.
- “학교 평가” 느낌이 드는 표현을 제거한다.
- “지원 우선순위”, “지원 필요 신호”, “공개데이터 기반 초기 진단”을 중심으로 쓴다.
- 버튼은 명확하게 쓴다.
  - 지도 보기
  - 학교 리포트 보기
  - 지원 우선순위 보기
  - 데이터 근거 보기

최종 검수:
npm run check:copy
npm run typecheck
npm run build

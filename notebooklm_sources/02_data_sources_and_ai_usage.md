## 교육 공공데이터와 보조 데이터 구분

교육 공공데이터는 1단계 지수 산출의 기본 근거이다.

- NEIS 학교 기본정보
- 학교알리미 공시자료
- 교육통계 및 교육청 공개자료

보조 데이터는 공개자료상 지원 신호를 보완하는 용도로만 사용한다.

- 지역 AI·SW 학습자원
- 공공 학습공간
- 지역 프로그램 접근 신호

AI 디지털교과서(AIDT)·학습관리시스템(LMS) 자료는 현재 총점에 넣지 않는다.
해당 자료는 제공될 경우 `확장진단 대기` 슬롯에서 별도 진단한다.

## 전처리 절차

1. 수집: 학교 기본정보와 공시자료를 학교명·주소 기준으로 결합한다.
2. 정제: 중복, 결측, 학교 유형을 점검한다.
3. 제외: 일반 비교가 어려운 특수목적 성격 학교 유형 2개를 제외한다.
4. 정규화: 비율·개수·접근 신호를 0-100 범위로 변환한다.
5. 신뢰도 분리: 결측이 많은 학교는 점수보다 현장 확인 우선으로 분류한다.
6. 리포트 생성: 학교별 보강 영역과 다음 행정 조치를 연결한다.

## 지원 소요 지수 산식

`지원 소요 지수`는 100점 만점이다.
점수가 높을수록 좋은 것이 아니라 지원 필요 신호가 크다는 뜻이다.

| 영역 | 배점 | 해석 |
| --- | ---: | --- |
| 교육 수요 | 25 | 학생 규모와 학급 규모 등 지원 수요 신호 |
| 교원·운영 여건 | 20 | 교원 배치와 운영 부담 신호 |
| 디지털·학습공간 기반 | 20 | 기기·무선망·학습공간 보강 신호 |
| AI·SW 학습기회 | 20 | 방과후·동아리·프로그램 접근 신호 |
| 지역 지원 접근성 | 15 | 학교 밖 자원 연계 필요 신호 |

## 직접지표와 대리지표

| 유형 | 예시 | 해석 |
| --- | --- | --- |
| 직접지표 | 학생 수, 교원 수, 학급 수, 시설 수, 프로그램 수 | 공개자료에서 직접 확인 가능한 항목 |
| 대리지표 | 교원 운영 여건, AI·SW 학습기회, 지역 접근성 | 직접자료가 부족한 영역의 보조 신호 |
| 추가자료 필요 | AI 수업 활용률, AI 디지털교과서(AIDT) 접속 로그, 학습관리시스템(LMS) 사용빈도, 교원 연수 이력 | 2단계 정밀 진단에 필요한 항목 |

## AI 활용 위치

규칙 기반 산식과 AI 보조 역할을 구분한다.

| 구분 | 역할 |
| --- | --- |
| 규칙 기반 산식 | 5개 영역 가중치, 총점 산출, 신뢰도 분류 |
| 공개자료 기반 산출 보조 | 지표 정규화 구조 검토, 결측 처리 기준 정리 |
| 공개자료 기반 분류 보조 | 유사 학교군 분류, 취약 요인 자동 라벨링 |
| 공개자료 기반 요약 보조 | 상세 리포트 설명문, 다음 행정 조치 문구 생성 |
| 지원 유형 매칭 | 예산 검토, 연수 배정, 프로그램 연계, 현장 확인 유형 연결 |

AI는 행정 결정을 대신하지 않는다.
AI는 공개자료 기반 산출·분류·요약을 보조한다.

## 생성형 AI 활용 기록 방식

PPT와 제출 문서에는 생성형 AI 활용을 다음 방식으로 기록한다.

| 항목 | 기록 내용 |
| --- | --- |
| 활용 도구 | 생성형 AI 기반 코드·문서 작성 보조 도구 |
| 입력 프롬프트 | 공공데이터 기반 MVP 개선, 익명화, 대시보드 설명력 강화 요청 |
| 산출물 | UI 문구, 지표 설명, 보고서 초안, 발표자료 원천 문서 |
| 검토 방식 | 사람이 산식, 데이터 출처, 표현 위험, 빌드 결과를 확인 |
| 책임 범위 | 최종 해석과 정책 판단은 사람이 수행 |

# 05. PPT Slide Source

## PPT 구성 원칙

- 15장 이내
- 12pt 미만 글자 사용 지양
- 한 슬라이드에 핵심 문장 1개
- 표는 5행 이하 중심
- 실제 학교명, 개인명, 구체 지역명 사용 금지
- 설명 화면보다 대시보드 화면 캡처 중심

## 1. 표지

### 제목

AI 교육격차 지도

### 핵심 문장

공개자료로 AI 교육 지원 필요 신호를 찾는 의사결정 MVP

### 화면 캡처

홈 또는 지도 상단

### 표·도식

없음

## 2. 문제 정의

### 제목

지원은 필요한데 우선순위가 보이지 않습니다

### 핵심 문장

교육청은 한정된 예산과 인력을 어디에 먼저 투입할지 빠르게 판단해야 합니다.

### 화면 캡처

홈 첫 화면 4개 카드

### 표·도식

문제 흐름: 공공데이터 분산 → 지원 판단 어려움 → 지도 기반 의사결정

## 3. 서비스 한 줄 정의

### 제목

학교 평가가 아니라 지원 소요 진단

### 핵심 문장

지원 소요 지수는 공개자료상 지원 필요 신호의 크기입니다.

### 화면 캡처

홈 핵심 배지

### 표·도식

배지 3개: 공개자료 기반, 지원 소요 진단, 제출용 익명화

## 4. 사용자와 의사결정

### 제목

교육청의 다음 결정을 돕습니다

### 핵심 문장

예산, 연수, 프로그램, 현장 확인을 학교별 지원 신호와 연결합니다.

### 화면 캡처

우선순위 카드

### 표·도식

| 지원 신호 | 행정 조치 |
| --- | --- |
| 디지털 기반 | 예산 검토 |
| 교원·운영 | 연수 배정 |
| 학습기회 | 프로그램 연계 |
| 결측 많음 | 현장 확인 |

## 5. 데이터 활용

### 제목

교육 공공데이터를 결합했습니다

### 핵심 문장

NEIS, 학교알리미, 교육통계, 교육청 공개자료를 1단계 지수의 근거로 사용했습니다.

### 화면 캡처

`/data` 활용 데이터 표

### 표·도식

교육 공공데이터와 보조 데이터 구분표

## 6. 산식 구조

### 제목

5개 영역으로 지원 소요를 산출합니다

### 핵심 문장

총점은 100점이며 점수가 높을수록 지원 필요 신호가 큽니다.

### 화면 캡처

`/data` 산식 카드

### 표·도식

| 영역 | 배점 |
| --- | ---: |
| 교육 수요 | 25 |
| 교원·운영 여건 | 20 |
| 디지털 기반 | 20 |
| AI·SW 학습기회 | 20 |
| 지역 접근성 | 15 |

## 7. AI 활용 위치

### 제목

AI는 산출·분류·요약을 보조합니다

### 핵심 문장

규칙 기반 산식과 AI 보조 역할을 분리했습니다.

### 화면 캡처

`/data` AI 활용 위치 카드

### 표·도식

규칙 기반 산식 / AI 보조 역할 2분할 도식

## 8. 지원 소요 지도

### 제목

지도에서 우선 지원 대상을 바로 봅니다

### 핵심 문장

TOP 3와 분포를 통해 먼저 볼 학교를 10초 안에 확인합니다.

### 화면 캡처

`/map` TOP 3와 지도 영역

### 표·도식

범례: 우선 지원, 보완 검토, 일반 모니터링, 현장 확인 우선

## 9. 우선순위 목록

### 제목

점수에서 다음 조치로 연결합니다

### 핵심 문장

각 학교 카드가 지원 신호, 신뢰도, 행정 조치를 함께 보여줍니다.

### 화면 캡처

`/priorities` 상위 카드

### 표·도식

3줄 구조: 왜 필요한가 / 무엇을 할 것인가 / 필요한 추가자료

## 10. 상세 리포트

### 제목

왜 이 점수인지 설명합니다

### 핵심 문장

5개 영역별 반영 점수와 판단 이유를 확인할 수 있습니다.

### 화면 캡처

`/schools/[id]` 점수 카드와 점수 근거

### 표·도식

없음

## 11. 신뢰도 C 처리

### 제목

자료가 부족하면 현장 확인이 먼저입니다

### 핵심 문장

신뢰도 C는 우선 지원에 섞지 않고 별도 분류합니다.

### 화면 캡처

지도 범례 또는 우선순위 현장 확인 영역

### 표·도식

점수 높음 + 신뢰도 C → 현장 확인 우선

## 12. 제출용 익명화

### 제목

식별정보는 제출 화면에서 제거합니다

### 핵심 문장

학교명, 주소, 연락처, 홈페이지, 실제 좌표를 익명화합니다.

### 화면 캡처

익명화된 지도와 상세 기본정보 영역

### 표·도식

| 항목 | 처리 |
| --- | --- |
| 학교명 | A초, B중, C고 |
| 지역 | 서울 A권역 |
| 좌표 | 서울 A권역 지도 배경 위 가상 배치 |
| 연락처 | 미표시 |

## 13. 한계와 확장

### 제목

AI 디지털교과서(AIDT)·학습관리시스템(LMS)는 2단계 확장진단입니다

### 핵심 문장

현재 총점에는 넣지 않고, 자료 제공 시 운영 안정성을 별도 진단합니다.

### 화면 캡처

상세 리포트의 확장진단 대기 영역

### 표·도식

접속 안정성 / 장애시간 / 사용 지속성 / 적용 범위

## 14. 정책 효과

### 제목

지원 자원 배분을 더 선명하게 합니다

### 핵심 문장

공개자료 기반 1차 진단으로 예산·연수·프로그램 지원 대상을 빠르게 좁힙니다.

### 화면 캡처

지도와 다음 조치 카드 조합

### 표·도식

데이터 → 지수 → 지도 → 조치 → 현장 확인

## 15. 마무리

### 제목

공공데이터에서 행정 조치까지

### 핵심 문장

AI 교육격차 지도는 학교를 줄 세우지 않고, 지원이 먼저 필요한 신호를 찾습니다.

### 화면 캡처

홈 또는 지도 첫 화면

### 표·도식

없음

# AI 교육격차 지도 프로그램 구성 및 사용 안내

작성 기준일: 2026-05-22  
검토 대상: `ai-education-gapmap-mvp`

## 1. 프로그램 개요

AI 교육격차 지도는 학교별 공공데이터를 결합해 AI 교육 준비도와 지원 필요 신호를 보여주는 MVP입니다. 핵심 목적은 학교를 평가하는 것이 아니라, 교육청·학교·지원기관이 어떤 지원을 먼저 검토할지 판단할 수 있도록 돕는 것입니다.

현재 구현은 서울 A권역 학교 데이터를 기준으로 구성되어 있습니다. 실제 지도 위에 학교 위치를 표시하고, 각 학교의 AI 교육 준비도 점수와 취약 요인을 함께 확인할 수 있습니다.

## 2. 핵심 사용자

| 사용자 | 주요 사용 목적 |
|---|---|
| 교육청 담당자 | 학교별 지원 필요 신호를 빠르게 파악하고 사업 우선 검토 대상을 좁힘 |
| 학교 관리자 | 자기 학교의 준비도 구성 요소와 보완 필요 영역을 확인 |
| 심사위원·검토자 | 공공데이터 연결, 지표 산식, 화면 흐름, 한계를 검토 |
| 정책·사업 기획자 | AI 교육 지원 사업의 대상 발굴과 사업 유형 분류에 참고 |

## 3. 현재 데이터 상태

현재 로컬 데이터 생성 결과는 다음과 같습니다.

| 항목 | 건수 |
|---|---:|
| NEIS 학교 기본정보 | 100 |
| 공공데이터포털 학교 위치 표준데이터 | 93 |
| 학교알리미 공시정보 | 96 |
| 정규화 학교 | 100 |
| 준비도 산출 | 100 |
| 지도 좌표 매칭 | 93 / 100 |

남은 데이터 경고는 없습니다.

학교알리미 API 표준 학교명과 맞지 않는 부설기관, 외국인학교, 학력인정 평생교육시설은 공식 공개자료 보조 근거를 별도로 기록해 반영합니다.

## 4. 전체 화면 구성

### 4.1 홈

경로: `/`

역할:

- 서비스의 목적과 핵심 메시지를 제시합니다.
- 분석 학교 수, 지원 우선 검토 수, 보완 필요 수, 데이터 경고 수를 요약합니다.
- 격차지도와 지원 우선순위 화면으로 이동합니다.

주요 문구:

> 학교를 평가하지 않고, 지원 우선순위를 찾습니다.

사용 방법:

1. 홈에서 전체 분석 규모를 확인합니다.
2. `교육격차 지도 보기`를 눌러 실제 지도 화면으로 이동합니다.
3. `지원 우선순위 보기`를 눌러 우선 지원 카드 목록을 확인합니다.

### 4.2 격차지도

경로: `/map`

역할:

- 실제 지도 위에 학교별 위치와 준비도 점수를 표시합니다.
- 좌표가 있는 93개 학교를 지도 마커로 보여줍니다.
- 우측 패널에서 준비도 점수가 낮은 학교를 우선 확인 목록으로 보여줍니다.
- 하단에서는 우선 지원 상세와 데이터 근거를 제공합니다.

지도 구현:

- 지도 라이브러리: Leaflet
- 지도 타일: OpenStreetMap
- 마커: 학교별 준비도 점수 표시
- 마커 색상:
  - 빨강: 집중 지원
  - 주황: 보완 우선
  - 파랑: 상대 상위

사용 방법:

1. `/map`으로 이동합니다.
2. 지도에서 점수 마커를 확인합니다.
3. 마커를 클릭해 학교명, 준비도 구간, 취약 요인, 상세 링크를 확인합니다.
4. 우측 `우선 확인` 목록에서 점수가 낮은 학교를 빠르게 살펴봅니다.
5. 하단 `우선 지원 상세`에서 취약 요인과 권장 지원 방향을 확인합니다.

### 4.3 학교 상세 리포트

경로: `/schools/[id]`

역할:

- 특정 학교의 AI 교육 준비도 점수와 구성 신호를 보여줍니다.
- 학교 기본정보, 학생 수, 교원 수, 주소, 홈페이지를 제공합니다.
- 점수 근거 표에서 각 지수의 개별 점수, 배점 반영 점수, 판단 이유를 확인할 수 있습니다.
- 추천 지원 방향을 카드 형태로 제공합니다.

사용 방법:

1. 지도 마커 또는 우선 확인 목록에서 학교를 선택합니다.
2. 준비도 점수와 취약 요인을 확인합니다.
3. 추천 지원 방향을 검토합니다.
4. `데이터 근거 보기`로 데이터·모델 설명 화면을 확인합니다.

### 4.4 지원 우선순위

경로: `/priorities`

역할:

- 지원 우선 검토 대상 학교를 카드 목록으로 보여줍니다.
- 각 학교별 주요 취약 요인과 지원 액션을 요약합니다.

사용 방법:

1. `/priorities`로 이동합니다.
2. 학교별 지원 유형을 확인합니다.
3. 카드의 액션 목록을 기준으로 필요한 지원 패키지를 검토합니다.

### 4.5 데이터와 모델

경로: `/data`

역할:

- 데이터 출처와 모델 산식을 설명합니다.
- 정규화 학교 수, 준비도 산출 수, 데이터 경고 수를 보여줍니다.
- 지수 산식의 배점 구조를 제공합니다.

준비도 지수 구성:

| 영역 | 배점 |
|---|---:|
| 학교 규모·수요 | 20 |
| 교원 여건 | 20 |
| 디지털 인프라 | 25 |
| AI·디지털 운영 신호 | 20 |
| 지역 지원 접근성 | 15 |

점수 해석:

| 점수 | 표시 |
|---:|---|
| 80 이상 | 준비도 높음 |
| 60~79 | 보완 필요 |
| 60 미만 | 지원 우선 검토 |

현재 데이터에서는 학교알리미 시설·동아리·방과후학교 공시와 공식 보조자료가 반영되어 학교별 점수 차이가 나타납니다. 공개자료상 운영 수치가 0인 학교는 해당 항목을 0점으로 유지합니다.

### 4.6 3분 시연

경로: `/demo`

역할:

- 발표나 심사용 3분 설명 흐름을 제공합니다.
- 문제, 데이터, 진단, 지원, 효과의 순서로 서비스 가치를 설명합니다.

사용 방법:

1. `/demo`로 이동합니다.
2. 다섯 단계 카드를 따라 서비스 흐름을 설명합니다.
3. `시연 시작`을 눌러 `/map`으로 이동합니다.

## 5. 데이터 파이프라인

### 5.1 수집

명령:

```bash
npm run fetch:public
```

역할:

- NEIS 학교 기본정보를 수집합니다.
- 공공데이터포털 전국초중등학교위치표준데이터를 수집합니다.
- 학교알리미 공시정보 `62`, `09`, `22`, `17`, `18`, `56`, `59`를 수집합니다.
- 공식 공개자료 보조 근거는 `data/source/public-supplements.json`으로 관리합니다.

생성 파일:

| 파일 | 설명 |
|---|---|
| `data/live/neis-schools.raw.json` | NEIS 학교 기본정보 |
| `data/live/school-location-standard.raw.json` | 학교 위치 표준데이터 |
| `data/live/schoolinfo-public.raw.json` | 학교알리미 학교 현황, 학생·교원·시설·동아리·방과후 공시정보 |
| `data/source/public-supplements.json` | API 표준 공시 밖 예외 학교의 공식 보조 근거 |
| `data/live/manifest.json` | 수집 결과와 경고 |

### 5.2 CSV 보조 자료 반영

명령:

```bash
npm run import:public-files
```

역할:

- `.env.local`에 지정한 CSV 파일 경로를 읽어 보조 데이터를 반영합니다.
- CSV 경로가 없으면 기존 API 출력물을 보존합니다.

사용 가능한 CSV 경로:

| 환경변수 | 용도 |
|---|---|
| `SCHOOLINFO_PUBLIC_CSV_PATH` | 학생 수, 교원 수, 학급 수 대체 자료 |
| `SCHOOL_FACILITY_CSV_PATH` | 시설 신호 |
| `DIGITAL_INFRA_CSV_PATH` | 디지털 인프라 신호 |
| `AI_PROGRAM_CSV_PATH` | AI·디지털 교육 운영 신호 |
| `TEACHER_TRAINING_CSV_PATH` | 교원 연수 신호 |
| `AI_CENTER_CSV_PATH` | 지역 AI 교육센터 자료 |
| `SW_AI_CAMP_CSV_PATH` | SW·AI 캠프 자료 |
| `PUBLIC_LEARNING_CENTER_CSV_PATH` | 공공 학습지원 자원 자료 |

## 6. API 키와 환경변수

실제 키는 `.env.local`에만 입력합니다. 문서나 저장소에는 키를 기록하지 않습니다.

현재 연결된 API:

| 환경변수 | 출처 | 용도 |
|---|---|---|
| `NEIS_API_KEY` | 나이스 교육정보 개방 포털 | 학교 기본정보 수집 |
| `SCHOOLINFO_API_KEY` | 학교알리미 Open API | 학교 현황, 학생 수, 교원 수, 학급 수, 시설, 동아리, 방과후 공시정보 수집 |
| `DATA_GO_KR_SERVICE_KEY` | 공공데이터포털 | 학교 위치 표준데이터와 위경도 수집 |

지도 관련:

- 현재 지도는 별도 지도 API 키 없이 Leaflet + OpenStreetMap으로 동작합니다.
- 네이버지도 또는 구글지도로 전환하려면 브라우저용 지도 API 키가 추가로 필요합니다.
- 네이버지도 전환 시 별도 환경변수 예시는 `NEXT_PUBLIC_NAVER_MAP_CLIENT_ID`입니다.
- 구글지도 전환 시 별도 환경변수 예시는 `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY`입니다.

## 10. 사용 시나리오

### 10.1 검토자 빠른 확인

1. `/map`을 엽니다.
2. 지도 위 학교 마커와 점수를 확인합니다.
3. 빨간색·주황색 마커가 많은 지역과 학교를 확인합니다.
4. 우측 `우선 확인` 목록에서 상위 학교를 누릅니다.
5. 학교 상세 리포트에서 취약 요인과 지원 방향을 확인합니다.
6. `/data`에서 산식과 데이터 한계를 확인합니다.

### 10.2 발표 시연 흐름

1. `/demo`에서 문제와 서비스 흐름을 설명합니다.
2. `/map`에서 실제 지도와 학교 마커를 보여줍니다.
3. 마커를 클릭해 팝업을 보여줍니다.
4. 학교 상세 페이지로 이동해 지원 방향을 설명합니다.
5. `/data`에서 공개데이터 기반 지표이며 학교 평가가 아니라 지원 신호임을 설명합니다.

### 10.3 운영자 데이터 갱신

1. `.env.local`의 API 키와 CSV 경로를 확인합니다.
2. `npm run prepare:data`를 실행합니다.
3. `npm run check:data`로 정규화 학교와 준비도 산출 수를 확인합니다.
4. `/map`에서 좌표 매칭 수와 지도 마커 표시 상태를 확인합니다.

## 11. 검토용 설명 문안

### 11.1 한 문장 소개

AI 교육격차 지도는 공공데이터를 기반으로 학교별 AI 교육 준비도와 지원 필요 신호를 시각화해, 교육청과 학교가 먼저 검토할 지원 방향을 찾도록 돕는 서비스입니다.

### 11.2 30초 설명

이 서비스는 NEIS, 학교알리미, 공공데이터포털의 학교 위치 데이터를 연결해 서울 A권역 학교별 AI 교육 준비도를 계산합니다. 실제 지도 위에 학교별 점수 마커를 표시하고, 점수가 낮은 학교는 우선 확인 목록과 상세 리포트에서 취약 요인과 권장 지원 방향을 확인할 수 있습니다. 목적은 학교를 평가하는 것이 아니라, 공개데이터로 지원 필요 신호를 정리하는 것입니다.

### 11.3 3분 발표 흐름

1. AI 교육은 확대되고 있지만 학교별 실행 여건은 다를 수 있습니다.
2. 이 MVP는 공공데이터를 연결해 학교별 준비도와 지원 필요 신호를 계산합니다.
3. 실제 지도 위에서 학교 위치와 준비도 점수를 함께 볼 수 있습니다.
4. 마커나 우선 확인 목록을 누르면 학교 상세 리포트로 이동합니다.
5. 상세 리포트는 취약 요인과 권장 지원 방향을 보여줍니다.
6. 이 결과는 학교 평가가 아니라 교육청과 학교의 지원 검토를 돕는 초기 진단입니다.

## 12. 현재 한계와 보강 방향

### 12.1 현재 한계

- 공개자료상 방과후학교·동아리 운영 수치가 0인 학교는 AI·디지털 운영 신호가 낮게 표시됩니다.
- 좌표는 93개 학교에 매칭되어 있고, 나머지 7개 학교는 실제 지도 마커에 표시되지 않습니다.
- 준비도 지수는 공개데이터 기반 초기 진단이므로 실제 수업 운영 수준을 직접 증명하지 않습니다.
- 네이버지도와 구글지도는 아직 적용하지 않았습니다. 현재는 키 없이 동작하는 OpenStreetMap 기반 실제 지도입니다.

### 12.2 보강 방향

- `AI_PROGRAM_CSV_PATH`에 AI 교육 운영 공개자료를 연결합니다.
- `DIGITAL_INFRA_CSV_PATH`와 `TEACHER_TRAINING_CSV_PATH`를 연결해 디지털 인프라와 교원 연수 신호를 보강합니다.
- 좌표 미매칭 7개 학교를 보완합니다.
- 브라우저용 네이버지도 또는 구글지도 API 키를 확보하면 지도 provider를 전환합니다.
- 학교별 실제 수업 운영 자료가 확보되면 준비도 지수의 설명력을 높입니다.

## 13. 검토 체크리스트

- `/map`에서 실제 지도 타일이 보이는가
- 지도 마커가 준비도 점수를 표시하는가
- 마커 클릭 시 학교 정보 팝업이 보이는가
- 우측 우선 확인 목록이 상세 페이지로 연결되는가
- 학교 상세 리포트에서 기본정보와 지원 방향이 보이는가
- `/data`에서 산식과 한계를 설명하는가
- `npm run check:data`가 통과하는가
- `npm run check:copy`가 통과하는가
- `npm run typecheck`가 통과하는가
- `npm run build`가 통과하는가

## 14. 보안 및 운영 주의사항

- API 키는 `.env.local`에만 보관합니다.
- 문서, README, Git 커밋, 화면 캡처에 실제 키를 노출하지 않습니다.
- 지도 타일은 OpenStreetMap을 사용하므로 외부 네트워크가 차단되면 지도 배경이 보이지 않을 수 있습니다.
- 데이터 갱신 후에는 반드시 `check:data`, `check:copy`, `typecheck`, `build`를 실행합니다.
